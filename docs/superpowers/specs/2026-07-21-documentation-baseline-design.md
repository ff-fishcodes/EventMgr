## 5. 当前已识别的重点差异与风险

文档必须如实记录以下事项：

| 类别 | 当前事实或差异 |
|------|----------------|
| 事件标识 | 设备事件使用 `deviceName-frameID-alarmField`；纯系统事件当前仍直接使用 `eventName`，与部分代码注释所称统一三段格式不完全一致 |
| 标识解析 | `BackendBridge` 和 `ExternalAPI` 按连字符拆分事件标识，设备名或报警字段包含连字符时可能无法正确解析 |
| 降级语义 | 当前降级改变 `effectiveLevel` 和前端颜色，但联动使用 `originalLevel`，因此降级不解除联动；这与原始需求存在冲突 |
| 屏蔽语义 | 屏蔽事件仍进入活跃表、写日志并执行联动；前端查询后再次过滤，不展示该事件 |
| 联动禁用 | 可按事件、动作和产生/消除侧独立禁用；配置仅保存在内存中 |
| 线程模型 | `EventManager` 和 `ConfigManager` 使用 `QMutex`，联动动作由四线程 `QThreadPool` 异步执行；当前后端并非无 Qt 依赖 |
| 并发边界 | `LinkageEngine` 的配置表和禁用集合没有显式锁保护，文档不得宣称其所有接口均线程安全 |
| 查询生命周期 | `EventManager::findEvent()` 返回内部对象指针，但锁在函数返回前释放，调用方不能据此获得稳定生命周期保证 |
| 对象生命周期 | `EventMgrModule::init()` 使用动态分配且没有对应释放入口，当前设计按进程生命周期持有这些对象 |
| 文档与示例 | README 的事件标识和部分调用示例仍沿用旧接口，需要同步到当前代码 |

这些事项在当前基线中属于“现状说明和风险”，不在本次工作中转化为代码修改。
